# ADR-012: 適応的スクリーンショット抽出とモーメンタムによるトークン圧縮

## ステータス

Accepted

## コンテキスト

動画分析パイプラインには 2 つの非効率があった。

1. **スクリーンショット抽出の固定間隔問題**: `extract_screenshots()` は `--interval` で指定した等間隔でフレームを抽出していた。集団戦やオブジェクト争奪のような密度の高いシーンでフレームが不足し、何も起きていないレーン戦フェーズでは無駄なフレームが生成されていた。

2. **match_context のトークン圧迫**: `_build_match_context()` は `player_stats` の全イベント（キル、デス、アイテム購入、レベルアップ、位置スナップショット等）をそのまま Gemini に渡していた。30 分超の試合では数百件のイベントになり、LLM のトークンを圧迫してチャンクあたりの分析精度が低下する懸念があった。

調査レポート「生成 AI を活用した LoL のゲーム動画分析」から、IaVNG の KIS（Key Interaction Segment）と LoL-MDC のモーメンタム・インパクトが既存パイプラインに直接適用可能と判断した。

## 決定

### Adaptive Interval（`--adaptive` オプション）

**OpenCV のヒストグラム比較による 2 パス方式で、シーン変化量に基づく適応的フレーム抽出を行う。**

- パス 1: 1 秒間隔でグレースケールヒストグラムの相関を計算し、活動量プロファイルを生成
- パス 2: 活動量が平均を超える区間は `base_interval / 3`、下回る区間は `base_interval * 2` でサンプリング
- `max_frames`（固定インターバルの 1.5 倍）で上限を制御し、トークン消費の爆発を防止
- `--adaptive` なしの場合は既存の固定インターバル動作を維持

### Momentum Impact（自動適用）

**ゴールド差タイムラインからピタゴラス勝率を近似計算し、勝率変動率（モーメンタム）が大きいタイムスタンプ周辺のイベントのみを match_context に残す。**

- ピタゴラス勝率: `P(t) = G1^α / (G1^α + G2^α)` where `G1 = base_gold + diff`, `G2 = base_gold - diff`, `α = 5.0`
- モーメンタム: 隣接フレーム間の確率差の移動平均（window=3）
- 上位 25 パーセンタイルのフレームインデックス ±30 秒のイベントのみ保持
- `gold_diff_timeline` が存在しない場合はフォールバック（全イベント保持）
- `match_data` 読み込み時に自動適用（オプトアウト不要）

### gold_diff_timeline の実装修正

従来 `riot_client.py` の `gold_diff_timeline` は全て 0 のプレースホルダーだった。Riot API のタイムラインフレームから味方チーム（participant_id 1-5 or 6-10）と敵チームのゴールド合計差分を正しく計算するよう修正した。

## 検討した代替案

### Adaptive Interval

| 代替案 | 却下理由 |
|---|---|
| フレーム差分の絶対値合計 | ヒストグラム比較のほうが照明変化やカメラ移動に頑健 |
| 1 パスの逐次判定 | 全体の変化量分布を見ないと閾値の設定が難しい |
| 機械学習ベースのシーン検出 | 依存が重く、このユースケースにはオーバーキル |

### Momentum Impact

| 代替案 | 却下理由 |
|---|---|
| XGBoost 等で精密な勝率予測 | 学習データの管理が必要で過剰 |
| 固定ルール（バロン・ドラゴンだけ残す） | 集団戦の重要度が反映されない |
| オプトインフラグ | 圧縮はデフォルトで有益、フォールバックもあるため常時適用が妥当 |

## 結果

### 実データ検証

**Momentum Impact**:
- Elise JUNGLE（35 分、敗北）で検証: 172 → 79 イベント（54.1% 削減）
- 9 分の -1,345 ゴールド差（勝率 0.2%）→ 12 分の +20（52%）の急変動を正しく検出

**Adaptive Interval**:
- 15 分リプレイ動画で固定 5 秒 vs adaptive を比較
- 高活動区間（9 分: +6 枚、13 分: +7 枚）でフレーム集中
- 低活動区間（14 分: -5 枚、3 分: -2 枚）でフレーム節約
- 解析内容比較: adaptive は戦闘の因果関係（レベル 2 → E 習得 → キル）をより細かく分解

### 既知の制限

- `α = 5.0` では勝率が 0%/100% に張り付きやすい（ゴールド差 ±1,500 以上で飽和）。α を下げると滑らかになるが、小さな変動がノイズとして検出されるトレードオフがある
- 2 パス方式は動画を 2 回読むため、処理時間が約 1.5 倍。15 分動画で activity scan に約 30 秒
- 解析内容の質の差は有意だが劇的ではない。重要場面の解像度に差が出る一方、静かな区間のカバレッジは固定インターバルのほうが均等
