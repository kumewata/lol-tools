# ADR-003: Gemini API による動画分析

## ステータス

Accepted

## コンテキスト

LoL の解説動画やリプレイ動画から、ゲームプレイに関する知識を構造化して抽出したい。動画には以下の情報源がある:

1. **音声/字幕** — 解説者の言語化された知識（解説動画）
2. **映像** — ゲーム画面のスクリーンショット（リプレイ動画）

これらを LLM で分析し、トピック・キーモーメント・改善提案を抽出する必要がある。

## 決定

**Google Gemini API をマルチモーダル LLM として採用し、テキストと画像の両方を統一的に処理する。**

主な設計:

- **Commentary モード**: YouTube 字幕をチャンクに分割し、各チャンクを Gemini で分析。最後に全チャンクを統合（synthesis）
- **Gameplay モード**: 定期的に抽出したスクリーンショットを Gemini に送信し、ゲーム状況を分析
- チャンク分割は `chunk_transcript()` で30秒間隔。長いセグメントは `end_ms` 基準で適切に分割
- 分析結果は Pydantic モデル（`ChunkAnalysis`, `AnalysisResult`）に構造化

## 理由

- **マルチモーダル対応**: Gemini はテキストと画像を同一プロンプトで処理でき、解説動画（字幕）とリプレイ動画（スクリーンショット）の両方に対応
- **音声文字起こし**: Gemini は音声ファイルの直接入力にも対応。ローカル動画の音声を m4a で抽出し、そのまま文字起こし可能（Whisper 不要）
- **日本語対応**: 日本語の解説動画・字幕を高品質に処理
- **コスト**: 個人利用レベルでは Gemini API の無料枠/低コスト枠で十分

## 影響

- Gemini API の `GOOGLE_API_KEY` が必要
- レスポンスの JSON パースに失敗するケースがあり、フォールバック処理が必要
- 画像のみの分析（Gameplay モード）は精度が低い。特に 360p では Champion 検出が 50% 以上の誤判定率。`--match-data` による試合データ連携が精度向上に必須（→ ADR-005）

## 追記

- 2026-04-08: ローカル動画の音声を Gemini で文字起こしする機能は削除した。現在の Commentary モードは YouTube 字幕取得を前提とし、ローカル動画は Gameplay モードのみサポートする。
